# 네이버 API MCP 서버 사용 시나리오

이 문서는 네이버 API MCP 서버(쇼핑인사이트 API + 검색 API)를 실제로 사용하는 다양한 시나리오 예시를 제공합니다.

---

## 📊 쇼핑인사이트 API 시나리오

### 시나리오 1: 패션 트렌드 분석

#### 상황
패션 의류 업체에서 최근 3개월간의 패션 트렌드를 분석하고 싶습니다.

#### 사용 방법
```
네이버 쇼핑인사이트에서 2024년 9월부터 12월까지 패션의류 카테고리의 주간 트렌드를 조회해주세요.
```

#### 예상 결과
- 패션의류 카테고리의 주간별 검색 비율 데이터
- 트렌드 변화 추이 분석
- 특정 시기(예: 가을, 겨울)의 트렌드 상승/하락 패턴

---

### 시나리오 2: 화장품 키워드 트렌드 분석

#### 상황
화장품 브랜드 마케터가 최근 인기 있는 화장품 키워드를 파악하고 싶습니다.

#### 사용 방법
```
2024년 10월부터 12월까지 화장품/미용 카테고리에서 "립스틱", "파운데이션", "스킨케어" 키워드의 월별 트렌드를 조회해주세요.
```

#### 예상 결과
- 각 키워드별 검색 트렌드 비교
- 어떤 키워드가 가장 인기 있는지 파악
- 월별 트렌드 변화 추이

### 시나리오 3: 경쟁 카테고리 비교 분석

#### 사용 방법
```
2024년 11월부터 12월까지 패션의류, 화장품/미용, 디지털/가전 카테고리의 주간 트렌드를 비교 조회해주세요.
```

#### 예상 결과
- 세 가지 카테고리의 트렌드 비교 데이터
- 각 카테고리의 상대적 인기도 파악
- 시기별 트렌드 변화 패턴


#### 예상 결과
- 이벤트 기간 동안의 일별 트렌드 변화
- 특정 날짜(예: 블랙프라이데이)의 트렌드 급등 패턴
- 이벤트 전후 트렌드 비교

## 주요 카테고리 ID 참고

- **패션의류**: `50000000`
- **화장품/미용**: `50000002`
- **디지털/가전**: `50000003`

## 사용 팁

### 쇼핑인사이트 API

1. **시간 단위 선택**
   - `date`: 일별 데이터 (최대 1개월)
   - `week`: 주별 데이터 (최대 3개월)
   - `month`: 월별 데이터 (최대 1년)

2. **기간 설정**
   - 너무 긴 기간은 데이터가 부정확할 수 있음
   - 일반적으로 1-3개월이 적절함

3. **키워드 선택**
   - 너무 구체적인 키워드는 데이터가 없을 수 있음
   - 일반적인 제품명이나 카테고리명 사용 권장

4. **비교 분석**
   - 여러 카테고리나 키워드를 동시에 조회하여 비교 가능
   - 트렌드 변화 패턴을 파악하기 좋음

5. **파라미터 형식 주의**
   - **분야별 트렌드 조회**: `categories`는 배열로 전달 (최대 3개)
   - **기기별/성별/연령별 조회**: `category`는 단일 문자열로 전달 (예: `"50000000"`)
   - **키워드별 트렌드 조회**: `keywords`는 배열로 전달 (최대 5개)
   - **키워드 기기별/성별/연령별 조회**: `keyword`는 단일 문자열로 전달 (예: `"립스틱"`)

6. **도구 선택 가이드**
   - 여러 카테고리 비교: `get-category-trends` 사용
   - 단일 카테고리의 세부 분석: `get-category-by-device/gender/age` 사용
   - 여러 키워드 비교: `get-keyword-trends` 사용
   - 단일 키워드의 세부 분석: `get-keyword-by-device/gender/age` 사용

### 검색 API

1. **검색어 선택**
   - 구체적인 검색어가 더 정확한 결과를 제공
   - 여러 검색어를 조합하여 사용 가능

2. **정렬 옵션 활용**
   - **블로그**: `date` (최신순) 또는 `sim` (정확도순)
   - **지식iN**: `point` (평점순)으로 인기 질문 파악
   - **쇼핑**: `asc`/`dsc` (가격순)으로 가격 비교

3. **필터 활용 (쇼핑 검색)**
   - `filter: naverpay`: 네이버페이 연동 상품만 조회
   - `exclude: used:cbshop`: 중고 및 해외직구 제외

4. **결과 개수 조정**
   - `display` 파라미터로 한 번에 조회할 결과 개수 조정 (최대 100개)
   - `start` 파라미터로 페이지네이션 가능 (최대 1,000)

## 주의사항

### API 제한사항

#### 쇼핑인사이트 API (DataLab)
- **카테고리 배열**: 최대 3개까지만 설정 가능 (`get-category-trends`)
- **키워드 배열**: 최대 5개까지만 설정 가능 (`get-keyword-trends`)
- **일일 호출 한도**: 하루 1,000회

#### 검색 API
- **일일 호출 한도**: 하루 25,000회
- **display**: 최대 100개
- **start**: 최대 1,000

### 파라미터 형식 주의
- **기기별/성별/연령별 API**: `category`는 **단일 문자열**로 전달 (예: `"50000000"`)
- **키워드 기기별/성별/연령별 API**: `keyword`는 **단일 문자열**로 전달 (예: `"립스틱"`)

### 기타
- 네이버 쇼핑인사이트 API는 최근 데이터만 제공합니다
- 과거 데이터는 제한적일 수 있습니다
- API 호출 제한이 있을 수 있으니 과도한 요청은 피하세요
- 검색 API는 실시간 검색 결과를 제공합니다
